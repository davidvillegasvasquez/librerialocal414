El comportamiento que describes se debe a cómo Python y Django gestionan los valores "falsy" (falsos) frente a un valor None explícito, combinado con cómo se envían los parámetros en las URL.
En resumen: Cuando envías ?var1= en la URL, get('var1', None) no devuelve None, devuelve una cadena vacía "".

Aquí está la explicación detallada:
1. ¿Por qué if var is not None: parece fallar?
Cuando haces self.request.query_params.get('var1', None):Si la URL no tiene var1 (ej: /api/users/), var es None.Si la URL tiene var1 pero vacío (ej: /api/users/?var1=), var es "" (una cadena vacía), no None.Por lo tanto, if var is not None: evalúa "" is not None, lo cual es True. Tu código entra en el bloque if creyendo que hay un valor cuando en realidad hay una cadena vacía.2. ¿Por qué if var: si funciona?if var: evalúa la "verdad" (truthiness) del objeto.Si var es None, if var: es False.Si var es "" (cadena vacía), if var: es False.Si var es "0", if var: es True (cuidado aquí).Si var es "hola", if var: es True.if var: es más estricto contra valores vacíos (cadenas, listas, etc.), mientras que is not None solo comprueba que no sea el objeto None.Soluciones RecomendadasDependiendo de qué consideres un valor "válido", utiliza una de estas opciones:A. Nota sobre DRF: Para manejar la validación de parámetros de consulta de forma más limpia, se recomienda utilizar el paquete django-filter o serializadores para validar query_params.

En Python y Django, una lista vacía [] o un QuerySet vacío arrojado por Django REST Framework (DRF) se evalúa como False (falso) en un contexto booleano.Esta característica, conocida como falsy, permite evaluar si una consulta no devolvió resultados de una manera directa y concisa.python# Ejemplo de evaluación de QuerySet vacío
queryset = MiModelo.objects.filter(campo="valor_que_no_existe")

if queryset:                # Evalúa a False si está vacío
    print("Hay datos")
else:
    print("El queryset está vacío") # Esto se ejecutará
Formas recomendadas de evaluar un QuerySet vacíoDirectamente en el if (La forma más Pythónica):Una lista vacía se evalúa como False, y una lista con elementos se evalúa como True.pythonresults = MiModelo.objects.filter(...)
if not results:
    print("No hay resultados")
Usando .exists() (La forma más eficiente para DRF):Para evitar evaluar todos los objetos en memoria, especialmente si el QuerySet es grande, es mejor usar .exists(). Esto devuelve un booleano (True/False) directamente de la base de datos.pythonif not MiModelo.objects.filter(...).exists():
    print("No existen registros")
Usando len() (Si ya has evaluado el queryset):Puedes verificar si la longitud es cero.pythonif len(queryset) == 0:
    print("El queryset está vacío")

      Objeto                            Evaluación Booleana               QuerySet con datos                         True

Nota: Una lista vacía es diferente de un valor None.
